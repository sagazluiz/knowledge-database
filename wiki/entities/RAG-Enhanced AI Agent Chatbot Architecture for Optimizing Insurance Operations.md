### Resumo Executivo

Este artigo propõe uma arquitetura de chatbot agente AI aprimorado com RAG (Retrieval-Augmented Generation) para otimizar operações de seguro de vida. A arquitetura integra o modelo de linguagem Ollama com a base de dados vetorial Supabase, alcançando uma fusão profunda da geração de linguagem e recuperação de conhecimento para cenários complexos de seguro.

### Componentes e Arquitetura Técnica

A arquitetura consiste em dois módulos principais: o módulo de dados e a interação com o chatbot. O módulo de dados serve como base do RAG, lidando com a aquisição, pré-processamento e armazenamento dos dados, enquanto habilita vetorização e recuperação semântica. A interação com o chatbot envolve a geração de embeddings, correspondência semântica, recuperação de documentos e geração de respostas.

### Glossário de Entidades

- **RAG (Retrieval-Augmented Generation):** Uma abordagem que combina modelos de linguagem grande com busca vetorial para gerar respostas mais ricas e contextualizadas.
- **LLMs (Large Language Models):** Modelos de linguagem grandes treinados em grandes quantidades de texto para entender e gerar linguagem natural.
- **Vector Database:** Uma base de dados que armazena documentos como vetores numéricos para busca semântica eficiente.
