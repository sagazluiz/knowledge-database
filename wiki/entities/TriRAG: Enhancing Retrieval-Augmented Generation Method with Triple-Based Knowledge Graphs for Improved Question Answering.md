"John went to the store and bought apples. He also bought bananas.", our method would extract triples such as "John", "went to", "store", "bought", "apples", and "bananas". This ensures that each triple carries complete semantic information, avoiding the problem of fixed-length segmentation where key information may be cut off by irrelevant content.

## C. Triple-Based Retrieval

In our TriRAG system, we employ a novel retrieval method based on structured knowledge graphs represented as triples. Traditional RAG methods typically involve retrieving text fragments from a vast corpus and then using these fragments to generate responses. However, this approach may lead to inefficiencies such as retrieving partially or completely irrelevant information, which can reduce the quality of generated responses.

Our method addresses this issue by converting raw text into structured triples that are easier to retrieve and more semantically relevant to the question. By focusing on triples rather than entire blocks of text, our approach ensures that only the most relevant information is considered during answer generation.

The retrieval process involves several steps:

1. **Embedding:** We use an embedding model to convert each triple into a vector representation. This allows us to compare the similarity between queries and triples in a more efficient manner.
2. **Query Embedding:** The query posed by the user is also converted into an embedding vector using the same embedding model.
3. **Similarity Calculation:** We calculate the cosine similarity between the query vector and each triple vector stored in our vector database. This step helps us identify which triples are most relevant to the query.
4. **Retrieval:** Based on the similarity scores, we retrieve the top-k most similar triples from the vector database.

By replacing the conventional text-based retrieval with a triple-based retrieval approach, TriRAG provides a more precise and efficient way for information retrieval, enhancing the accuracy of responses generated by LLMs in multiple-choice question-answering tasks.

## D. Augmented Text Generation

After retrieving the most relevant triples based on the query, our system uses these triples to augment the input text provided to the LLM during response generation. This step is crucial as it allows the LLM to consider additional context and information when generating its response.

The augmented text consists of the original question followed by the retrieved triples. For example, if the user asks "Who wrote 'To Kill a Mockingbird'?", our system might retrieve the triple "<Harper Lee>, <wrote>, <To Kill a Mockingbird>" and augment the input text as follows:

"Who wrote 'To Kill a Mockingbird'? Harper Lee wrote To Kill a Mockingbird."

By providing this augmented context, we enable the LLM to generate more accurate and relevant responses by considering the additional information provided in the retrieved triples.

## B. Implementation Details

We implement TriRAG using the Hugging Face Transformers library [19] and employ the Sentence-BERT model [20] for embedding triples and queries into vectors. We use the Faiss library [21] to efficiently search for similar triples in our vector database.

For the LLM component, we experiment with several leading models including Gemma, Llama, and ChatGPT variants. We fine-tune these models on the Textbook Question Answering dataset using the AdamW optimizer with a learning rate of 3e-5 and a batch size of 16 for 3 epochs.

## VI. CONCLUSION AND FUTURE WORK

In this paper, we introduced TriRAG, an enhanced Retrieval-Augmented Generation method that integrates structured knowledge graphs of semantic triples to improve the performance of large language models on multiple-choice question-answering tasks. Our experimental results demonstrate that TriRAG outperforms traditional RAG methods across several leading LLMs including Gemma, Llama, and ChatGPT variants.

In future work, we plan to explore the following directions:

1. **Dynamic Triple Selection:** Currently, our method retrieves a fixed number of top-k triples based on similarity scores. In future work, we aim to develop a dynamic triple selection mechanism that adaptively determines the optimal number of retrieved triples based on query complexity and relevance.
2. **Knowledge Graph Expansion:** While our current approach focuses on extracting triples from existing knowledge bases, we plan to investigate methods for actively expanding and updating knowledge graphs by incorporating new information from external sources or user interactions.
3. **Multimodal Knowledge Integration:** We aim to extend TriRAG to support multimodal knowledge graphs that integrate textual, visual, and other modalities, enabling more comprehensive question answering and retrieval capabilities.

In conclusion, our work demonstrates the potential of integrating structured knowledge graphs with Retrieval-Augmented Generation methods to enhance LLM performance in multiple-choice question-answering tasks. By focusing on semantic triples and employing efficient retrieval techniques, TriRAG provides a powerful framework for improving information retrieval accuracy and contextual relevance in LLMs.
